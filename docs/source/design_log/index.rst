Design Log
==========

Transcripts of the docs phase interview for each SDD run. Each entry captures
the full conversation between the developer and the Docs Skill — the questions
asked, the answers given, and the decisions made.

Design log entries are tagged with C4 element IDs and requirement IDs so they
are searchable by component when revisiting past decisions:

.. code-block:: bash

   grep -r "auth_component" docs/source/design_log/
   grep -r "REQ-AUTH-001"   docs/source/design_log/

Design log content is **reference-only** — it is never fed back into pipeline
context. It exists so the rationale behind decisions is never lost.
